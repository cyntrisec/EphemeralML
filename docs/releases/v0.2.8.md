# EphemeralML v0.2.8

**Released:** 2026-02-19
**Commit:** `d7b9e3c`
**Tag:** [v0.2.8](https://github.com/cyntrisec/EphemeralML/releases/tag/v0.2.8)

## What changed

- **Strict compliance collect** (`--strict` flag): `ephemeralml-compliance collect` now fails
  closed when required evidence types (attestation, manifest) are missing from the bundle.
- **Require destroy evidence** (`--require-destroy-event` flag): `ephemeralml-verify` can now
  mandate that the receipt contains destroy evidence with at least one action.
- **Sidecar evidence saving**: The GCP client binary now writes `attestation.bin` and
  `manifest.json` sidecar files alongside the receipt, enabling complete compliance bundles.
- **6 new conformance tests** (CT-020 through CT-025): strict-mode rejection, complete bundle
  assembly, auto-discovery, full baseline pass, hash-mismatch detection, enriched destroy evidence.
- **E2E golden path hardened**: `mvp_gpu_e2e.sh` step [5] requires destroy evidence, step [6]
  always passes `--strict`, metadata version updated.

## Security defaults

| Control | Default | Override |
|---------|---------|----------|
| Destroy evidence required | Off | `--require-destroy-event` |
| Strict evidence collect | Off | `--strict` |
| Baseline compliance profile | 16 rules | `--profile baseline` |
| Receipt max age | 3600s | `--max-age N` |

The E2E golden path enforces both `--require-destroy-event` and `--strict` by default.

## Known limits

- **GPU path**: Requires `a3-highgpu-1g` instances with H100 CC; availability is quota-dependent
  per region. CPU path (`c3-standard-4`) validated in this release.
- **Attestation sidecar**: Only produced in GCP mode; mock mode returns `None` for
  `boot_attestation_b64` and `model_manifest_json`.
- **FRESH-001 rule**: Receipt age check uses wall-clock comparison; clock skew between enclave
  and verifier is not compensated.
